# Agent Fiscal Autonomy Pack for OpenClaw

Direct Base USDC agent-commerce proof layer for OpenClaw agents.

This public repo is a teaser and trust surface. It shows strict JSON crypto strategy research artifacts, source-citation discipline, safety boundaries, and the buyer-agent commerce flow. It is not the paid package and does not contain raw course files or the paid release ZIP.

## What this is

A source-grounded fiscal-autonomy research pack for OpenClaw agents that need to evaluate whether crypto strategy research can help justify operating costs such as VPS, model calls, APIs, tools, and other agent skills.

The default workflow is research and paper-trading only.

## What this is not

- Not financial advice.
- No guaranteed profit.
- No direct live trading.
- No exchange order placement.
- No raw private vault files.
- No private Google Drive vault links.

## Machine-readable proof examples

The repo exposes three strict JSON proof artifacts:

- `wiki_term_pass_v1`: source-linked glossary/concept card.
- `indicator_card_v1`: technical-marker card with regimes and failure modes.
- `setup_playbook_v1`: setup checklist with invalidation and required evidence.

See `examples/` and `schemas/`.

## Buyer-agent flow

1. Inspect the proof artifacts in this repo.
2. Open a seven-field Audit Request: `https://github.com/egoriklok/agent-fiscal-autonomy-pack/issues/new?template=audit_request.md`.
3. Include agent type, monthly operating-cost target, intended use case, current data/tooling, max loss or risk budget, desired proof report, and safety-boundary acceptance.
4. Receive a quote and Base USDC invoice only after qualification is clear.
5. Pay exact USDC amount on Base and provide tx hash.
6. Receive entitlement and restricted delivery instructions after verification.

## Safety boundary

All strategy outputs are research artifacts. Any missing `source_ids`, entry trigger, invalidation, or risk rule keeps strategy action blocked. Runtime exchange execution is outside this product.
