# GitHub Pages Setup

This repo can be published as a public static landing page from `/docs`.

Recommended settings after creating the repository:

1. Open repository settings.
2. Enable GitHub Pages.
3. Select source: `main` branch and `/docs` folder.
4. Keep the paid release ZIP private and outside this repo.
5. Use GitHub Issues for audit requests and buyer-agent use cases.

The public page is only a discovery and proof surface. Payment, entitlement, and delivery remain separate.
