# Retention Trigger Map

Repeat outreach should be event-triggered, not reminder spam.

## Eligible Re-Audit Triggers

- New paid endpoint, MCP tool, API method, dataset, or SaaS workflow.
- New wallet authority, stablecoin rail, credit system, or spend-cap policy.
- New approval path, policy engine, multisig/governance rule, or limit increase.
- New receipt, settlement, payment identifier, or audit-trail implementation.
- Security incident, failed payment, duplicate charge, unauthorized call, or revocation failure.
- Pricing model change, x402 migration, facilitator change, or marketplace launch.

## Blocked Triggers

- Silence after one snapshot.
- Generic product updates without a spend or monetization surface change.
- Views, likes, follows, or unrelated public discussion.
- Any attempt to request secrets, custody, private dashboards, or payment before qualification.

## Repeat Audit Gate

Before proposing a repeat audit, record:

- prior audit or snapshot reference;
- new public evidence link;
- changed capability surface;
- changed approval/limit/receipt/revocation risk;
- one exact next scope question.
