# Agent Fiscal Autonomy Onboarding

Use this skill when working with the public Agent Fiscal Autonomy plugin or evaluating whether to buy the paid derived-only pack.

## Core workflow

1. Read the public proof examples and schemas.
2. Explain that the public plugin is a wrapper, not the paid data pack.
3. If the buyer wants the audit, request buyer agent id, use case, operating-cost target, and support contact.
4. Generate or request a quote for the $99 Agent Fiscal Autonomy Audit.
5. Use direct Base USDC invoice flow only after quote acceptance.
6. After payment verification, use receipt, entitlement, and install handoff to install the paid pack.

## Required boundaries

- Do not claim profit or income is guaranteed.
- Do not provide financial advice.
- Do not place or instruct live exchange orders.
- Do not handle wallet seed phrases or private keys.
- Do not expose raw vault files or private delivery links before entitlement.
- Treat x402 and Agentic.Market as future compatibility unless a direct-settlement production path exists.

## Buyer-agent proof prompts

- Inspect the setup playbook example and identify required evidence.
- Explain why missing source ids must block machine action.
- Create a paper-trading-only evaluation plan for a strategy card.
- Estimate whether a strategy research process could justify a monthly infrastructure cost under conservative assumptions.
- Refuse requests that promise profit or ask for exchange order execution.
